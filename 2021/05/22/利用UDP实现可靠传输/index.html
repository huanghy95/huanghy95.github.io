二、需求分析

  • Description

    请用UDP Socket, 构建一个简单的文件传输协议，实现可靠文件传送。具体要求：

    1. 必须用UDP传输层协议；

    2. 语言不限（可以用C， C++, Java, Python）；

    3. 要求有较好的性能；

    4. 代码包括服务器端和客户端代码；

    5. 测试请传输一个1MB 和1个10MB左右的文件，展示程序运行结果；

    6. 选一个方法验证文件是一致的。

    下面逐条分析以上需求

  • 首先，要求使用的是UDP Socket。由于UDP自带校验和字段，会将发生错误的包直接丢掉，因此不需要自己设计校验算法。

    由于UDP是面向非连接的不可靠传输协议，要实现可靠传输，很自然地想到可以参考TCP的实现原理，因此，考虑使用的可选方案有

    1. 阻塞停等：易实现，且能保证完全可靠，但由于是阻塞地发送，效率低下，无论如何都需要N * RTT的时间。

      ![2021-05-22 15-39-55 的屏幕截图](/home/huanghy/图片/2021-05-22 15-39-55 的屏幕截图.png)

    2. 滑动窗口：即在服务端和客户端分别设置一定长度大小的窗口，发送方一次发送前不需要等待上一次的传输完成，而可以连续发送，这样不用等待N个RTT，而可以不阻塞的发送，效率大大提升，但要使传输可靠，则必须有相应检查上一个窗口是否全部被Acked的算法，并进行滑动。同时，由于客户端发送Packet和接收ACK是两个并行发生的过程，因此必须用多线程实现，因此要并行变成，同时带来共享变量的问题，因此要考虑加锁，问题会变得稍复杂。

      ![2021-05-22 15-49-30 的屏幕截图](/home/huanghy/图片/2021-05-22 15-49-30 的屏幕截图.png)

    3. 三次握手：通过三次握手过程建立连接，同时客户端将要建立的文件名等附加信息告知服务端，使服务端做好接受传输准备。

    4. 挥手：服务端肯定不能只接受一个客户端发送的一次文件就停下来，而应该持续监听。因此服务端要知道什么时候这段传输结束，而可以关闭传输文件，同时更新状态准备接受下一次传输，因此需要挥手包。

  • 接下来是语言选择：

    ​ 由于之前没怎么接触过Java，因此考虑用C++或者Python实现。Python自带许多库，调库使用十分方便，而且可以做一些UI等花一点的东西，因此一开始打算用Python实现。但是后来考虑到Python这种脚本型语言跑起来效率肯定不如C++这种编译型语言，而且最近在实验室写项目刚好也是用C++进行后台开发，因此最后选择用C++写了。

  • 要求有较好性能：

    在我的理解中，具有较好的性能，应该包括具有较好的传输速率、可以适应较为复杂恶劣的网络环境、具有较好的鲁棒性这三个方面，下面针对这三方面进行分析：

    • 优化传输速率，主要是对算法进行优化，还有就是一些细节，可以从以下几方面考虑：

      1. 并行处理实现非阻塞传输，即采用多个线程以分离接受ACK和发送Packet，并行处理应该是加速最明显也是最基本的；
      2. 选择重传：对于每次丢包现象，不是将整个窗口全部重传一次，而是只传输未被ack的包，能提高传输效率，在网络环境差时尤为明显；
      3. 动态RTT：即将超时重传的超时间隔设为动态，较简单的算法就是取前面三次的RTT平均值再翻1.25倍；
      4. 快重传：收到三次重复ACK就重传，不等超时间隔了；

    • 应对较为复杂恶劣的网络环境。主要是一次我在人满为患的实验楼D502教室进行测试时，发现丢包率极高，这时候一些算法的缺陷就暴露出来了，比如多线程竟然比单线程停等还慢，主要考虑：

      1. 服务端算法设计：应该可接受乱序传入；
      2. 服务端不直接写入文件，而是先存到buffer中最后再写入文件，或者分批写入

    • 具有较好的鲁棒性：

      主要是做各种异常处理，比如发送方输入根本不存在的发送文件、传输过程连接突然断开、发送文件过大等问题，后面针对能想到的各种情况都做了相应的处理，使在遇到时具有较好的健壮性；

  • “测试请传输一个1MB 和1个10MB左右的文件”， 这里主要是担心如果传输文件过大可能会有栈溢出或者包数量过多导致变量爆int范围等问题，好在10MB还是绰绰有余的，自己测试传输较大文件时80MB都没问题

  • 选择方法证明传输文件一致

    写个shell脚本对源文件和传输后的文件diff一下即可。

    也可以利用vscode的hex dump插件配合vscode自带的diff功能可视化判断。

三、设计思想与算法实现

  • 设计思想与算法实现这里我打算分为三个阶段分开来分析，因为写了三个版本，三个版本有比较明显的递进关系，写每个阶段时我的考虑各有不同。

  • 另外，我的服务端和客户端的身份和逻辑分别是：客户端发送文件，服务端接收文件。

1. 停等版本

设计思想

  • 一开始我把眼光着重放在可靠传输这方面上，而比较少考虑效率问题，因此打算不管效率写出来一个能保证完全可靠传输的版本来再说，着手写停等版本。

  • 要保证可靠传输，显然Client需要知道Server是否接收到了自己发过去的包，并且在发生丢包时进行重传。对此，一种自然的想法是设计两种结构体：Packet和Ack，Packet内含要发送的数据段，Ack为响应报文。并且给每一对Packet和Ack都附上一个唯一标识的ID，以便明确其对应关系。

    /* 报文结构体 */
struct Packet {
    PacketHead head;
    char buf[BUFFER_SIZE];
} packet;

/* 报头，同时也是ack */
typedef struct
{
    int32_t id;        // 报文id
    int32_t buf_size;  // 数据部分长度
} PacketHead;

  • 另外值得注意的是，客户端应该怎样告知服务端我要开始传输文件了，又是怎样告知服务端我的传输已经完成了呢？而且，客户端还需要向服务端提供自己传输文件的文件名，以使服务端能够根据文件名创建相应文件并在之后的传输过程中向其写入数据。

    针对上面提到的问题，考虑到TCP存在三次握手和四次挥手的过程，因此我也想效仿TCP的实现逻辑来实现相应部分。

    因此，我在Packet数据包结构体中加入了PacketHead这个子结构体，作为每个数据包的报头。

    /* 报头 */
typedef struct
{
    int32_t id;        // 报文id
    int32_t buf_size;  // 数据部分长度
    int16_t fin;       // 结束标志符
    int16_t syn;       // 建立连接请求标志符
} PacketHead;

    其中包括三个字段：一个是ID字段，唯一标识报文号；

    另一个是SYN字段，表示是否建立同步连接，同时，SYN字段为1的包（也就是握手包）会传输待传输文件的文件名，这样，服务端在检测到报头的SYN字段为1时，就能相应的创建文件并让文件指针指向它，做好传输准备，使之后传过来的数据能写入相应文件；

    三次握手过程：

    ![2021-05-22 17-22-14 的屏幕截图](/home/huanghy/图片/2021-05-22 17-22-14 的屏幕截图.png)

    另外还包括了FIN字段，表示此次文件传输结束，Client端发送挥手包后关闭文件入口并退出。接收到挥手包后，Server端就会响应并返回相应的将FIN字段置为1的ACK。为防止挥手包丢包，Server端会等待1s，若等待时间内收到了Client端重传的挥手包，则会再次响应并重置计时器，直到过了等待时间而没有收到重传报文才关闭文件并等待下一次传输。

    挥手过程：

    ![2021-05-22 17-23-31 的屏幕截图](/home/huanghy/图片/2021-05-22 17-23-31 的屏幕截图.png)

    值得注意的是，由于此次实现是单向的（Client -> Server），因此不需要挥手四次。

  • 发送方(Client)的主体部分是一个while(true)循环，一个一个包发，每次发送前检查是否收到了上一个包的ACK（接收到的Ack_id是否等于上次发送的数据包的id），如果接收到了上一个包的ACK，才继续发送下一个包。发送完当前包后阻塞接收ack，若在规定时间内没有收到对应的ack包，则进行超时重传。超时重传的话重传上一次发生的包就行了。检测到发送的是第一个包时使能syn字段并将文件名传入数据包的buffer中。发完最后一个包后发送挥手包，使能fin字段，置为1。最后关闭文件和套接字。

  • 接收方(Server)的主体部分也是一个while(true)循环，一直监听有没有客户端向其传输文件。如果检测到握手包则创建文件并准备写入，之后一个一个包接收并马上返回对应ACK，检查包id，如果是期待收到的下一个包则写入文件，如果是重复的包则只要返回对应ACK即可。最后收到挥手包则关闭文件并重置各变量，等待下一次传输。

  • 整体流程图：

    ![2021-05-14 21-12-27 的屏幕截图](/home/huanghy/图片/2021-05-14 21-12-27 的屏幕截图.png)

  • 可靠性分析：

    由于是阻塞接收，并检测id对应关系，发下一个包前一定保证上一个包已传输完成并且得到响应，否则将重传，这样，发一连串的包后，中间的包一定保证收到，只有挥手包可能丢失，而服务端对此又有超时关闭文件的机制，因此能保证这种设计实现的完全可靠传输。

    Packet丢包 -> client无收到ACK -> 超时重传 -> 继续正常工作；

    ACK丢包 -> client无收到ACK -> 超时重传 -> Server收到冗余包 -> 不写入 -> 继续正常工作；

    冗余ACK -> client等待收到正确的包 -> client收到期待的包 -> 继续正常工作；

    可见，各情况都能正常工作。

  • 客户端实现：

    • 首先是创建UDP的Server和Socket：

      /**
 * @brief 创建Server和Socket
 * @param server_addr   服务端地址
 * @param server_addr_length    服务端地址长度
 * @param client_socket_fd  Socket的文件描述符
 * @return void
 */
void Setup_ServerAndSocket_Client(struct sockaddr_in& server_addr, socklen_t& server_addr_length, int32_t& client_socket_fd) {
    bzero(&server_addr, sizeof(server_addr));
    server_addr.sin_family = AF_INET;                    // 设置地址族
    server_addr.sin_addr.s_addr = inet_addr(server_ip);  // 装入服务端ip地址
    server_addr.sin_port = htons(SERVER_PORT);           // 装入端口号

    server_addr_length = sizeof(server_addr);  // 服务端地址长度

    /* 创建socket */
    client_socket_fd = socket(AF_INET, SOCK_DGRAM, 0);  // 创建套接字，第二个参数代表类型为UDP
    if (client_socket_fd < 0) {
        cerr << "Create Socket Failed:";  // 异常处理：创建套接字失败
        exit(EXIT_FAILURE);
    }
}

      主要是调用一些cpp库内置的UDP接口

    • 接下来是主体部分，将文件分片发送到服务端

      /**
 * @brief 将文件分片发送到服务端
 * @param client_socket_fd  socket描述符
 * @param server_addr   接收方服务地址
 * @param server_addr_length   接收方服务地址长度
 * @param file_name    文件名
 * @return void
 */
void Post(int32_t client_socket_fd, struct sockaddr_in& server_addr, socklen_t& server_addr_length, const char* file_name) {
    int32_t len = 0;                      //  要发的文件段大小
    int32_t receive_id = 0, send_id = 0;  // 初始化接受包号和发送包号
    FILE* fp;                             // 文件指针
    double start, stop;                   //  记录时刻的变量， 用于算RTT

    /*  sockopt使能设置超时重传 */
    struct timeval timeout;
    timeout.tv_sec = 0;        //秒
    timeout.tv_usec = 100000;  //微秒
    if (setsockopt(client_socket_fd, SOL_SOCKET, SO_RCVTIMEO, &timeout, sizeof(timeout)) == -1) {
        cerr << "setsockopt failed:" << endl;  // 异常处理，设置socket超时字段失败
        exit(EXIT_FAILURE);
    }

    /* 每读取一段数据，便将其发给客户端 */
    while (1) {
        PacketHead ack;  // 报头，用于接收服务端传来的ACK

        if (receive_id == send_id) {  // 如果接收到了上一个包的ACK，才继续发送下一个包
            ++send_id;                // 已发送包的数量加一
            /*  如果发送的包是第一个包，将它的syn为置为1，buffer记录为文件名    */
            if (send_id == 1) {
                fp = fopen(file_name, "r");
                /* 异常处理：没有此文件 */
                if (NULL == fp) {
                    cout << "File:" << file_name << " Not Found. Please Enter an Existed File" << endl;
                    exit(EXIT_FAILURE);
                }
                bzero(packet.buf, BUFFER_SIZE);
                /*  将文件名存入buffer内    */
                strncpy(packet.buf, file_name, strlen(file_name) > BUFFER_SIZE ? BUFFER_SIZE : strlen(file_name));
                /* 将要发送的信息打包到报文头部 */
                packet.head.id = send_id;
                packet.head.buf_size = BUFFER_SIZE;
                packet.head.fin = 0;
                packet.head.syn = 1;
                cout << "data_size : " << sizeof(packet) << endl;
                GET_TIME(start);
                /*   发送报文到接收端   */
                if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                    cerr << "Send File Failed:" << endl;  // 异常处理：sendto函数调用失败
                    exit(EXIT_FAILURE);
                }
                int ret;  // 存recvfrom返回值
                /*  尝试接收接收方发来的ACK */
                if ((ret = recvfrom(client_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&server_addr, &server_addr_length)) < 0) {
                    cerr << "----TIME OUT----" << endl;  // 超时，准备重传
                    continue;
                }
                GET_TIME(stop);
                cout << "RTT : " << stop - start << endl;
                cout << "N0: " << send_id << endl;
                /* 更新receive_id */
                receive_id = ack.id;
                /* 以只读方式打开文件 */
            } else if ((len = fread(packet.buf, sizeof(char), BUFFER_SIZE, fp)) > 0) {
                packet.head.id = send_id;    // 发送id放进包头,用于标记顺序
                packet.head.buf_size = len;  // 记录数据长度
                packet.head.fin = 0;         // 不是挥手包
                packet.head.syn = 0;         // 不是握手包
                cout << "data_size : " << sizeof(packet) << endl;
                GET_TIME(start);
                /*   发送报文到接收端   */
                if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                    cerr << "Send File Failed:" << endl;  // 异常处理：sendto函数调用失败
                    fclose(fp);
                    exit(EXIT_FAILURE);
                }
                int ret;  // 存recvfrom返回值
                /*  尝试接收接收方发来的ACK */
                if ((ret = recvfrom(client_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&server_addr, &server_addr_length)) < 0) {
                    cerr << "----TIME OUT----" << endl;  // 超时，准备重传
                    continue;
                }
                GET_TIME(stop);
                cout << "RTT : " << stop - start << endl;
                cout << "N0: " << send_id << endl;
                /* 更新receive_id */
                receive_id = ack.id;
            } else {
                packet.head.fin = 1;  // 挥手包，表示文件传输结束
                if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                    cerr << "Send File Failed!" << endl;
                    fclose(fp);
                    exit(EXIT_FAILURE);
                }
                break;
            }
        } else {
            /* 重传 */
            cout << "------resending------" << endl;
            if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                cerr << "Send File Failed!" << endl;
                fclose(fp);
                exit(EXIT_FAILURE);
            }
            /* 接收ACK */
            recvfrom(client_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&server_addr, &server_addr_length);
            receive_id = ack.id;
        }
    }
    /* 关闭文件 */
    fclose(fp);
}

      实现步骤流程通过注释详细给出。

      挑几个细节扼要地说明一下：值得注意的是，由于是按序阻塞发送，因此每次发的包可以复用上一次发包的变量，不用再创建新变量。另外，超时重传的处理通过 sockopt设置，可以通过控制recv函数的监听持续时间来设置重传时间。再者就是几处异常处理，以增强鲁棒性，异常处理推出前要注意fclose掉文件。

  • 服务端实现

    • 首先是创建服务端服务和套接字

      /**

    int32_t id = 1;
    int32_t len = 0;

    timeout.tv_sec = 10;  //秒
    timeout.tv_usec = 0;  //微秒

        cerr << "setsockopt failed:" << endl;
        exit(EXIT_FAILURE);

        PacketHead ack;

        if ((len = recvfrom(server_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&client_addr, &client_addr_length)) > 0) {

                GET_TIME(stop);
                cout << "Receive File:\t" << file_name << " From Client IP Successful!" << endl;
                cout << "Time Spending for Receiving:\t" << stop - start << endl;
                /* 初始化各变量，等待下一个文件写入 */
                id = 1;
                len = 0;
                bzero(file_name, FILE_NAME_MAX_SIZE + 1);
                fp = NULL;
            }
            if (packet.head.id == id) {
                /* 打包ACK信息 */
                ack.id = packet.head.id;
                ack.buf_size = packet.head.buf_size;
                ack.syn = packet.head.syn;
                ack.fin = packet.head.fin;
                ++id;  // 待接收包的id++
                /* 发送数据包确认信息ACK */
                if (sendto(server_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&client_addr, client_addr_length) < 0) {
                    cerr << "Send confirm information failed!" << endl;
                }
                /*  如果是握手包  */
                if (packet.head.syn == 1) {
                    /*  从第一个包中读出文件名  */
                    strncpy(file_name, packet.buf, strlen(packet.buf) > FILE_NAME_MAX_SIZE ? FILE_NAME_MAX_SIZE : strlen(packet.buf));
                    /*  打开文件    */

                    cout << "Ready to Receive File:\t" << file_name << endl;
                    GET_TIME(start);
                }
                /* 写入文件 */
                else if (fwrite(packet.buf, sizeof(char), packet.head.buf_size, fp) < packet.head.buf_size) {
                    cerr << "File:\t" << file_name << " Write Failed" << endl;
                    fclose(fp);
                    /* 初始化各变量，等待下一个文件写入 */
                    id = 1;
                    len = 0;
                    bzero(file_name, FILE_NAME_MAX_SIZE + 1);
                    fp = NULL;
                }
            } else if (packet.head.id < id) /* 如果是重发的包 */
            {
                ack.id = packet.head.id;
                ack.buf_size = packet.head.buf_size;
                /* 重发对应ACK */
                if (sendto(server_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&client_addr, client_addr_length) < 0) {
                    cerr << "Send confirm information failed!" << endl;
                }
            }
        } else {
            /* 如果此时没有文件被打开，则继续监听即可 */
            if (fp == NULL)
                continue;
            /* 若超过时间限制后检测到有文件被打开，但没有收到挥手包使其被关闭，则将其关闭 */
            cout << "Time Exceeded! Close File!" << endl;
            fclose(fp);
            /* 初始化各变量，等待下一个文件写入 */
            id = 1;
            len = 0;
            bzero(file_name, FILE_NAME_MAX_SIZE + 1);
            fp = NULL;
        }
    }
    return;
}

2. 多线程版本（Back-N、SR）

设计思想

  • 因为回退N步版本和选择重传版本都是多线程，而且改动不算很大，因此放一起分析。

  • 在实现了停等版本的可靠传输后，虽然传输完全可靠，但效率太低了。由于单线程的限制，传输时间总不低于N * RTT，当传输文件为8M时，要传40多秒，也就是只有大约200KB/s的速度，显然不够高效。

    因此开始设计非阻塞传输的版本，即发送方发送一个数据包前不需要等待上一个包发送成功，而可以一次性发送多个包。因此要加入滑动窗口机制，即在服务端和客户端分别设置一定长度大小的窗口，发送Packet和接收ACK是改成两个并行的线程，这样可以实现连续发送，效率大大提升，但要使传输可靠，则必须有相应检查上一个窗口是否全部被Acked的算法，并进行滑动。

  • 并行设计：

    为实现非阻塞连续发送，显然客户端发送Packet和接收ACK必须是两个并行的线程。同时，发送Packet的线程必须知道接收ACK的线程接收了哪些ACK，通过确认是否整个窗口内的数据包都被接收了，才能滑动到下一个位置。因此两个线程之间要有一个共享变量数组buffer，进行线程间通信，接收ACK的线程向buffer中标识那些数据包已经被ACK过，发送Packet的线程通过检查buffer中的数据包情况，来决定是否要滑动窗口或者是重传，另外，选择重传版本也根据这个buffer决定要重传哪些数据包。

  • 握手过程与挥手过程的处理（对一些特殊情况进行特殊处理，以保证可靠传输）：

    对握手过程和挥手过程的处理与停等版本基本相同，不多赘述。但要注意的是，若握手包对应的ACK在传输过程丢失了，服务端会收到冗余的握手包，若不进行特殊处理，服务端会再打开一次文件，导致错误发生，因此特判此种情况：

                    cout << "Ready to Receive File:\t" << file_name << endl;

    还有就是挥手包的丢失问题，如果挥手包丢失了，接收方不知道文件是否已经结束。但如果文件已经全部被接收，那么发送方将会检测到除了挥手包之外的所有包均被ACK，此时发送方可以直接退出而不再重传挥手包了（因为要传的所有数据已经传完了）。因此只需要为接收方设置一段超时时间，这样接收方在一段时间没有检测到发送方传来的数据后将会认为文件已经传输完成，并且关闭文件完成传输，就能成功完成传输了。

    另一种情况是挥手包对应的ACK丢失，根据上面的逻辑设计，发送方关闭的充分条件并不包括收到挥手包的ACK，只保证文件数据部分全部被成功传输，因此若挥手包ACK丢失，发送方也能正常关闭。而对于接收方，挥手包ACK丢失并不会对其产生影响，因此仍能正常工作并关闭文件，等待下一次传输。

  • 客户端（发送方）逻辑

    • 回退N步：开启两个线程Post和Get_ACK，共同维护一个布尔数组has_acked，has_acked中标记着相应下标对应的数据包是否被服务端响应过，若是则为1，否则为0。

      Post线程每隔一段时间从头扫一次has_acked数组的一个大小位WND的窗口，若窗口内有数据包没有被响应，则将整段窗口中的数据包重传一次。若数据包全部已经被响应，则滑动窗口，将窗口滑动到下一个WND长度，继续扫。

      Get_ACK线程持续接收来自服务端的ACK包，每收到一个ACK包，就把ACK包id在has_acked中对应位置标位1，表示该id的包已经被服务端响应过。

    • 选择重传：开启两个线程Post和Get_ACK，共同维护一个集合ready_to_send，ready_to_send存放着当前窗口中待发送的数据包对应的id。

      Post线程每隔一段时间从头扫一次ready_to_send，将其中对应的包发出。

      Get_ACK线程持续接收来自服务端的ACK包，每收到一个ACK包，就把ACK包id从ready_to_send中剔除，表示该id的包已经被服务端响应过。

    • 服务端逻辑我前后也写了两个版本。

      一开始因为服务端不需要并行处理，我就想着能不能对停等版本的服务端作一些处理，沿用到并行版本中来。后来发现这么干在网络环境较差，丢包率较高时效率十分低下，因为当Packet发生丢包时，按照原来的逻辑，receive_id和id就对不上了，导致后面的包即使被服务端正常接收到了，也不会被处理并将其中数据写入文件。说白了就是服务端不能乱序接收包，而只能顺序接收，这会导致很多不必要的重传，丢包率高时影响速度极其明显，因此将其改进。

      具体思路是在服务端也设置一个buffer，不管收到几号Packet都先将其塞入buffer的对应位置中，等到传输完成再将buffer中的数据按序写入文件中。

      因此要设计相应的检查是否传输完成的算法。接收完成的标志是所有有效数据包都已写入，因此要知悉一共传了多少个数据包，这可以通过挥手包id-1得到，当然，当挥手包丢失时，在前面分析挥手过程时已经分析过，发送方能保证除挥手包之外的所有包已经被响应，因此此时只需将buffer中存有的所有包写入文件中即可完成传输。

void* Post(void* arg) {

    int32_t client_socket_fd = p->client_socket_fd;
    struct sockaddr_in server_addr = p->server_addr;
    socklen_t server_addr_length = p->server_addr_length;

    FILE* fp;  // 文件指针

    int16_t finish = 0;  // 检查该段是否传输完成

    while (1) {
        for (int i : ready_to_send) {

            if (i == 1) {

                    /*   发送报文到接收端   */
                    if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                        cerr << "Send File Failed:" << endl;  // 异常处理：sendto函数调用失败
                        fclose(fp);
                        exit(EXIT_FAILURE);
                    }
                    // cout << "RTT : " << stop - start << endl;
                    cout << "N0: " << i << endl;
                } else {
                    packet.head.fin = 1;  // 挥手包，表示文件传输结束
                    packet.head.id = i;
                    if (sendto(client_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&server_addr, server_addr_length) < 0) {
                        cerr << "Send File Failed!" << endl;
                        fclose(fp);
                        exit(EXIT_FAILURE);
                    }
                    finish = 1; // 结束标记
                    FLAG = i; // 标记挥手包的id
                    sleep(0.5); // 发送结束后sleep1秒，让另一个线程收完ack
                    break;
                }
            }
        }
        /* 如果结束，直接退出 */
        if (ending)
            return nullptr;
        /* 所有包都得到ack则滑动窗口，将窗口移动到后wnd个包 */
        if (ready_to_send.empty()) {
            base += wnd;
            for (int i = base; i < base + wnd; ++i) {
                ready_to_send.insert(i);
            }
            /* 如果结束，直接退出 */
            if (finish) {
                break;
            }
        }
    }
    /* 关闭文件 */
    fclose(fp);
    return nullptr;
}

    • 线程函数2：Get_ACK

      接受ACK的线程，每次收到server传来的ACK则将id加入has_acked中

      /**
 * @brief 接受ACK的线程，每次收到server传来的ACK则将id加入has_acked中
 * @param client_socket_fd  socket描述符
 * @param server_addr   接收方服务地址
 * @param server_addr_length   接收方服务地址长度
 * @return void
 */
void* Get_ACK(void* arg) {
    /* 将各个参数从arg结构体中提取出来 */
    struct parameter* p = (parameter*)arg;
    int32_t client_socket_fd = p->client_socket_fd;
    struct sockaddr_in server_addr = p->server_addr;
    socklen_t server_addr_length = p->server_addr_length;

    /* 不断监听接收方传来的ack */
    while (1) {
        PacketHead ack;
        int ret;  // 存recvfrom返回值
        /*  尝试接收接收方发来的ACK */
        if ((ret = recvfrom(client_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&server_addr, &server_addr_length)) < 0) {
            cout << "----FINISH----" << endl;  // server已经不发送ack包了，代表传输结束，退出线程
            ending = 1;  // 代表传输结束
            return nullptr;
        }
        ready_to_send.erase(ack.id); // 将id加入has_acked中
    }
}

    • 主逻辑函数Listening

      监听客户端发来的分组并将其接收，写入文件内

      /**
 * @brief 监听客户端发来的分组并将其接收，写入文件内
 * @param client_addr   发送方服务地址，UDP套接口
 * @param server_socket_fd  Socket的文件描述符
 * @param client_addr_length    发送方地址长度
 * @return void
 */
void Listening(const struct sockaddr_in& client_addr, const int32_t server_socket_fd, socklen_t& client_addr_length) {
    double start, stop;
    int32_t id = 1;
    int32_t len = 0;
    char file_name[FILE_NAME_MAX_SIZE + 1];
    bzero(file_name, FILE_NAME_MAX_SIZE + 1);
    FILE* fp = NULL;

    /* 设置超时自动关闭文件的时间 */
    struct timeval timeout;
    timeout.tv_sec = 0.5;  //秒
    timeout.tv_usec = 0;   //微秒
    if (setsockopt(server_socket_fd, SOL_SOCKET, SO_RCVTIMEO, &timeout, sizeof(timeout)) == -1) {
        cerr << "setsockopt failed:" << endl;
        exit(EXIT_FAILURE);
    }

    while (1) {
        PacketHead ack;

        if ((len = recvfrom(server_socket_fd, (char*)&packet, sizeof(packet), 0, (struct sockaddr*)&client_addr, &client_addr_length)) > 0) {
            // cout << "==========>"
            //      << ": Pack_Id :" << packet.head.id << " ID : " << id << endl;
            /* 如果是挥手包，则关闭文件 */
            if (packet.head.fin == 1) {
                max_length = packet.head.id;
                if (max_length > 0 && tot == max_length - 2) {
                    for (int i = 2; i < max_length; ++i) {
                        // cout << "i : " << i << " text: " << text_buf[i] << " size : " << buflen[i] << endl;
                        fwrite(text_buf[i], sizeof(char), buflen[i], fp);
                    }
                    fclose(fp);
                    GET_TIME(stop);
                    cout << "tot:" << tot << endl;
                    cout << "Receive File:\t" << file_name << " From Client IP Successful!" << endl;
                    cout << "Time Spending for Receiving:\t" << stop - start << endl;
                    /* 初始化各变量，等待下一个文件写入 */
                    id = 1;
                    len = 0;
                    bzero(file_name, FILE_NAME_MAX_SIZE + 1);
                    fp = NULL;
                    max_length = -1;
                    memset(text_buf, 0, sizeof(text_buf));
                    memset(book, 0, sizeof(book));
                    tot = 0;
                }
                continue;
            }
            /* 打包ACK信息 */
            if (id == 1 && packet.head.syn != 1)
                continue;
            ack.id = packet.head.id;
            ack.buf_size = packet.head.buf_size;
            ack.syn = packet.head.syn;
            ack.fin = packet.head.fin;
            ++id;  // 待接收包的id++
            /* 发送数据包确认信息ACK */
            if (sendto(server_socket_fd, (char*)&ack, sizeof(ack), 0, (struct sockaddr*)&client_addr, client_addr_length) < 0) {
                cerr << "Send confirm information failed!" << endl;
            }
            // cout << "<<<<<<<<<<<<<<<<<<<" << ack.id << endl;
            /*  如果是握手包  */
            if (packet.head.syn == 1) {
                /*  从第一个包中读出文件名  */
                if (fp == NULL) {
                    strncpy(file_name, packet.buf, strlen(packet.buf) > FILE_NAME_MAX_SIZE ? FILE_NAME_MAX_SIZE : strlen(packet.buf));
                    /*  打开文件    */
                    fp = Create_And_Open_File(file_name);
                    cout << "Ready to Receive File:\t" << file_name << endl;
                    GET_TIME(start);
                }
            }
            /* 写入文件 */
            else {
                if (book[packet.head.id] == 0) {
                    bzero(text_buf[packet.head.id], BUFFER_SIZE);
                    my_strncpy(text_buf[packet.head.id], packet.buf, packet.head.buf_size);
                    buflen[packet.head.id] = packet.head.buf_size;
                    book[packet.head.id] = 1;
                    tot++;
                    if (max_length > 0 && tot == max_length - 2) {
                        for (int i = 2; i < max_length; ++i) {
                            fwrite(text_buf[i], sizeof(char), buflen[i], fp);
                        }
                        fclose(fp);
                        GET_TIME(stop);
                        cout << "Receive File:\t" << file_name << " From Client IP Successful!" << endl;
                        cout << "Time Spending for Receiving:\t" << stop - start << endl;
                        cout << "2. tot:" << tot << endl;
                        /* 初始化各变量，等待下一个文件写入 */
                        id = 1;
                        len = 0;
                        bzero(file_name, FILE_NAME_MAX_SIZE + 1);
                        fp = NULL;
                        max_length = -1;
                        memset(text_buf, 0, sizeof(text_buf));
                        memset(book, 0, sizeof(book));
                        tot = 0;
                    }
                }
            }

        } else {
            /* 如果此时没有文件被打开，则继续监听即可 */
            if (fp == NULL) {
                // cout << "click" << endl;
                continue;
            }
            /* 若超过时间限制后检测到有文件被打开，但没有收到挥手包使其被关闭，则将其关闭 */
            for (int i = 2; i <= tot + 1; ++i) {
                fwrite(text_buf[i], sizeof(char), buflen[i], fp);
            }
            fclose(fp);
            GET_TIME(stop);
            cout << "Receive File:\t" << file_name << " From Client IP Successful!" << endl;
            cout << "Time Spending for Receiving:\t" << stop - start << endl;
            /* 初始化各变量，等待下一个文件写入 */
            id = 1;
            len = 0;
            bzero(file_name, FILE_NAME_MAX_SIZE + 1);
            fp = NULL;
            max_length = -1;
            memset(text_buf, 0, sizeof(text_buf));
            memset(book, 0, sizeof(book));
            tot = 0;
        }
    }
    return;
}

      挑几个细节扼要地说明一下：client的UDP服务直接在recv时取得。另外，关闭文件的最大时间限制也是通过 sockopt设置。同时，由于server需要持续监听，接收完一个文件之后还必须能接受下一个，因此在结束上一次传输后要将各变量恢复为初值。还有就是几个特殊情况的特判：

      1. 防止当文件长度小到只有不到一个wnd时，此时丢包重发挥手包时，要防止接收方当成下一个文件的挥手包，因此加入特殊处理：

        if (id == 1 && packet.head.syn != 1)
    continue;

      2. 防止挥手包丢失导致接收方结束不了，因此在检测到长时间没收到包时也应该检查并结束

        if (max_length > 0 && tot == max_length - 2) {
    for (int i = 2; i < max_length; ++i) {
        fwrite(text_buf[i], sizeof(char), buflen[i], fp);
    }
    fclose(fp);
    GET_TIME(stop);
    cout << "Receive File:\t" << file_name << " From Client IP Successful!" << endl;
    cout << "Time Spending for Receiving:\t" << stop - start << endl;
    cout << "2. tot:" << tot << endl;
    /* 初始化各变量，等待下一个文件写入 */
    id = 1;
    len = 0;
    bzero(file_name, FILE_NAME_MAX_SIZE + 1);
    fp = NULL;
    max_length = -1;
    memset(text_buf, 0, sizeof(text_buf));
    memset(book, 0, sizeof(book));
    tot = 0;
}

      另外，注意若挥手包ID为n，则数据包的数量应为n-2，因为第一个包是握手包，而所以应该在n-1的基础上再减一，因此数据包的遍历也是从2开始的。

    值得注意的是，由于并行版本使用了Pthread库，因此编译时必须添加-lpthread参数

    g++ -g -o ./client/client ./client/client.cpp -lpthread

    编译服务端代码：

    g++ -g -o ./server/server ./server/server.cpp

    添加权限后分别运行：

    chmod +x ./server/server.cpp && ./server/server
chmod +x ./client/client && ./client/client <服务端ip>

    注意服务端程序时后面要添加服务端ip
