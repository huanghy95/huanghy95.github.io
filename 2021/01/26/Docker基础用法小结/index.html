Docker基础用法小结

Created2021-01-26|Updated2021-09-26
|Post View:

总结一些很基础的Docker用法

入门级Docker

基本概念

镜像：类，img，打包好的小环境

容器：实例，虚拟机

仓库：放镜像的地方

基本命令

  • docker pull ：从镜像仓库拉取现有镜像

    e.g. docker pull nginx

  • docker images： 查看当前有哪些docker镜像

  • docker run <镜像名>：将镜像跑成容器

    e.g. docker run -d -p 80:80 nginx

    参数：

    • -d：后台运行
    • -p：外部端口映射到内部端口
    • -v：映射目录

  • docker ps：查看当前正在跑的docker容器

  • docker exec -it <容器ID> <终端，如bash>：进入容器（命令行形式）

  • docker commit <容器id> <镜像名>：将容器打包成镜像，docker images里会多出新建的镜像

  • docker build -t <镜像名> <Dockerfile所在文件>：将Dockerfile建成镜像

  • docker save <镜像名> > <tar文件名>：将镜像打包成tar文件

  • docker load < <tar文件名>：读取tar文件到镜像

  • docker rm -f <容器id>：删除正在运行的容器

  • docker rmi <镜像名>：删除保存的镜像

reference：https://www.bilibili.com/video/BV1R4411F7t9?spm_id_from=333.999.0.0

Dockerfile书写

基本语句

  • FROM：基于某个原有的镜像
  • WORKDIR：指定接下来shell语句的运行目录
  • COPY：将指定容器外目录复制到指定容器内目录里
  • ADD：除了有COPY的功能外，还可以使用网络资源url
  • RUN：容器构建时运行的shell脚本
  • CMD：容器运行时运行的shell脚本
  • ENTRYPOINT：类似CMD
  • EXPOSE：指定暴露端口
  • VOLUME：映射目录
  • ENV：指定容器环境变量
  • ARG：构建时生效
  • LABEL：指定元数据，标识作用
  • ONBUILD：继承镜像才会运行父镜像里ONBUILD的内容
  • STOPSIGNAL：指定容器使用什么信号，一般指定信号名
  • HEALTHCHECK：检查容易的健康状态
  • SHELL：如指定linux为/bin/bash，windows为cmd

reference：https://www.bilibili.com/video/BV1k7411B7QL?spm_id_from=333.999.0.0

组织Docker项目

Docker通信

  • 同一docker网卡下，不同容器之间进行通信可以直接通过内网ip

  • docker run --link：可以在创建一个容器时将另一个容器的ip映射到特殊变量中

    e.g docker run -dit --link mynginx:mynginx alpine

    将mynginx容器ip映射到mynginx变量上，这样在创建的alpine容器中就可以通过mynginx变量访问mynginx容器

  • docker-compose：多级link，组织多容器项目
